# Supabase setup (backend)

This file explains how to wire Supabase into the backend of this project.

1) Create a Supabase project
- Go to https://app.supabase.com and create a new project.
- Note the project URL (e.g. https://xyz.supabase.co) and configure a strong password for the database.

2) Create tables
- Use the SQL editor or Table Editor to create a `products` table with the required columns.

Example SQL:

```sql
create table products (
  id bigint generated by default as identity primary key,
  title text,
  price numeric,
  description text,
  image_url text,
  created_at timestamptz default now()
);
```

3) Row Level Security and policies
- Enable RLS for `products` if you want to restrict writes.
- Example policy for public read:

```sql
-- Allow read for everyone
create policy "Public read" on products
for select
using (true);
```

4) Get API keys
- In Project Settings → API copy the `URL` and `SERVICE ROLE` key.
- Keep the `SERVICE ROLE` key secret (server-only). Use the anon key in the frontend for public calls.

5) Backend environment variables
Create `backend/.env` (DO NOT commit) with:

```
SUPABASE_URL=https://xyz.supabase.co
SUPABASE_SERVICE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

6) Install dependency and run backend

```powershell
cd backend
npm install @supabase/supabase-js dotenv
# start your server (example)
node index.js
```

7) Example routes
- `GET /api/products` — returns list of products
- `POST /api/products` — create product (server should verify admin)

8) Frontend wiring
- Add `frontend/.env` with VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY
- Example `.env`:

```
VITE_SUPABASE_URL=https://xyz.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

9) Security notes
- Never commit service_role keys.
- Use RLS policies. Verify JWTs in backend for protected operations.
